The following changes were made in this release:

0.1.8.0
- Fixed writing of C# volatile fields.
- Refactored code writers to share a common base code visitor and improved writing for varying configurations.
- Preserve header comments (i.e. do not attach to using/import statements) and by default surround with a "Header" region.
- Added a != operator for condition expressions.
- Fixed some behavior in multiple attribute (i.e. nested) sorting.
- Allow configuration of writing end region names.
- Fixed parsing of properties with multiple index parameters.
